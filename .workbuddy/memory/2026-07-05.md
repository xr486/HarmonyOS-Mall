## 下午修复工作（16:00 - 16:20）

### 问题 1：ArkTS 编译错误 - UpdateCartBody 缺少 selected 字段
**根因**：`UpdateCartBody` 类只有 `quantity` 字段，缺少 `selected`
**修复**：在 `HttpUtil.ets` 中添加 `selected: boolean = false`

### 问题 2：购物车页面数据为空（数据库有数据但页面不显示）
**根因**：前后端字段名不匹配
- 后端 `CartItem` 实体使用 **`checked`** 字段
- 前端 `CartItem` 接口使用 **`selected`** 字段
- API 返回 JSON 中是 `"checked": true`，前端期望 `"selected": true`，导致选中状态始终为 undefined

**修复方案**：
1. 修改 `Cart.ets` 模型，添加 `checked?` 可选字段兼容后端
2. 修改 `CartService.getCartItems()` 添加字段映射逻辑
3. 修改 `UpdateCartBody.selected` → `UpdateCartBody.checked` 与后端一致

### 问题 3：加减按钮显示为蓝色圆点
**根因**：鸿蒙默认 `Button` 组件样式为蓝色胶囊形状
**修复**：商品详情页和购物车页的 +/- 改用 `Text` 组件 + 自定义样式 + 边框

### 问题 4：加入购物车没有成功提示
**根因**：`ProductDetailPage.addToCart()` 方法静默处理结果
**修复**：导入 `promptAction`，成功/失败/异常分别显示不同 Toast

## 购物车数据为空问题修复与 UI 排版再优化（17:00 - 17:15）

### 问题：购物车页面再次为空（数据库有数据但页面不显示）
**根因分析**：
- 后端购物车接口 `/api/v1/cart` 需要 token 鉴权
- 前端 ArkTS 类型断言可能导致后端 `checked` 字段在运行时丢失
- 之前 `CartService.getCartItems()` 使用 `ApiResponse<CartItem[]>` 直接访问 `raw.checked`，在 ArkTS 严格模式下可能不可靠

**修复方案**：
1. 在 `CartService.ets` 中新增 `RawCartItem` 接口（已导出）
2. 改用 `ApiResponse<Object[]>` 接收后端原始数据
3. 使用 `JSON.stringify` + `JSON.parse` 重新解析，得到 `Record<string, Object>`
4. 通过 `raw['checked']` 索引方式安全读取字段，避免类型断言问题
5. 在 `CartPage.loadCart()` 和 `CartService.getCartItems()` 中添加日志，便于排查

### 购物车页面 UI 排版优化
**用户反馈**：根据截图，购物车缺少选择框、加减按钮、编辑删除等功能，排版不够美观

**优化内容**：
1. **商品卡片**：白色圆角卡片，每项包含左侧 Checkbox + 商品图片 + 名称/规格/价格 + 数量调整器/删除按钮
2. **选择框**：支持单选和全选，使用 `Checkbox.onClick()` 避免递归触发
3. **数量加减器**：改为方形边框样式，减号在数量=1时置灰，加号正常
4. **编辑模式**：导航栏右侧"编辑"/"完成"切换，编辑模式下显示删除按钮和底部批量删除

### 后端接口路径修正

## CartTab 选择框与布局二次修复（18:00 - 18:20）

### 用户反馈
用户截图指出：
1. 全选后商品选择框没有同步勾选
2. 商品名称出现在图片上方，不在图片右侧
3. 只显示 1 个商品，但合计显示 "共 3 件"、¥29997
4. 加减数量功能异常

### 修复内容
1. **Checkbox 状态同步**：`CartTab` 中 Checkbox 从 `onClick` 改为 `onChange` 事件，并移除 `group` 属性，避免状态不联动
2. **布局优化**：商品卡片 `Row` 使用 `VerticalAlign.Top` 顶部对齐；内部 `Column` 使用 `justifyContent(FlexAlign.SpaceBetween)` 让名称在上、价格/数量在下，替代 `Blank().layoutWeight(1)` 的不稳定布局
3. **ForEach key 防重**：key 函数从 `item.cartId` 改为 `item.cartId + '_' + index`，防止 cartId 重复时只渲染一个
4. **加减数量修复**：`updateCartItemQuantity()` 改用 `UpdateCartQuantityBody`（只有 `quantity` 字段），避免发送 `checked:false` 误重置后端选中状态
5. **Tab 切换不刷新修复**：`CartTab` 通过 `@Link @Watch('onRefreshKeyChange') refreshKey` 接收父组件刷新信号，每次切到购物车 Tab 时 `Index.onTabChange` 增加 `cartRefreshKey` 触发自动刷新
6. **自顶向下紧凑布局**：去除标题栏 `top: 12` padding 和列表顶部 padding，让商品卡片紧贴标题栏，减少页面顶部空白
6. **选择框/数量刷新不同步**：`CartTab` 和 `CartPage.ets` 的 `ForEach` key 增加 `selected` 和 `quantity` 字段，确保选中状态或数量变化后重新渲染对应卡片
